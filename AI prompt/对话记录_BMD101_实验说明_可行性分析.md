### 1. USB版BMD101心电资料 讲解
- 硬件：NeuroSky BMD101 SoC (AFE + 16-bit ADC + DSP)
- 通信协议：ThinkGear Packet (0xAA 0xAA sync, checksum, 57600 baud)
- 数据项：POOR_SIGNAL(0x02), HEART_RATE(0x03), RAW_WAVE(0x80, 512Hz)
- 使用方式：胸部/手背测量
- 配套资料：Arduino源码, C#上位机, FT232驱动, 安卓/iOS SDK

### 2. BMD101_py 内容讲解
- Python端：bmd101plotter.py (实时绘图), bmd101recorder.py (绘图+CSV记录)
- ESP32端：esp32_bmd101.ino (解析+滤波), esp32_bmd101_filtered.ino (解析+滤波+OLED)
- 滤波链路：0.5Hz HP → 50Hz LP → 60Hz Notch (IIR Butterworth)
- C++滤波器库：IIRFilter, FIRFilter, BiQuadFilter, Cascade, SOSFilter

### 5. Python 内感受视觉训练程序 (2026-05-30)

根据实验说明制作了完整的Python实验程序：

**文件位置**: `ecg_intervention/ecg_biofeedback_training.py`

**程序架构**:
- `RPeakDetector` — 实时R波检测：带通滤波(0.5-30Hz) + 自适应阈值 + 200ms不应期
- `BMD101Worker` — 串口读取线程，解析 ThinkGear 协议，计算心率
- `SimulatedWorker` — 模拟模式（无硬件时测试用，60-80 BPM 随机心率）

- `MainWindow` — 全屏GUI：深色主题，大字体中文显示
- `PortDialog` — 启动对话框：被试编号、训练次数、串口选择、模拟模式

**实验流程**:
- 启动 → 填写被试编号/训练次数/串口 → 2秒等待连接
- **组块1**: 反馈阶段(24试次) → 无反馈阶段(24试次) → 5分钟休息（开合跳）
- **组块2**: 同上 → 5分钟休息
- **组块3**: 同上 → 完成
- 每个试次 N=2~5 随机，按键窗口 [200ms, 450ms]

**数据保存**: `ecg_intervention/recordings/{被试编号}_session{次数}_{时间戳}/`
- ecg_raw.csv, r_peaks.csv, trials.csv, summary.json

**关键技术细节**:
- 使用 scipy.signal.lfilter + zi 实现逐样本实时带通滤波
- 多状态管理：trial_active / trial_feedback / phase_summary / rest / complete
- 最大试次超时 15s，防止R波丢失导致卡死
- 跨线程信号槽通信（QThread + QObject worker）

**依赖**: PyQt5, pyserial, numpy, scipy

## 11. 第六轮：串口读取 Bug 修复 — BMD101 无法读取心率 (2026-06-02)

### 11.1 问题
连接心电硬件后，模拟模式关闭，但程序完全没有读取心率数据（HR 显示 "-- BPM"），R 波检测事件也未触发。

### 11.2 根本原因：校验和算法错误（一补码 vs 二补码）

`_parse_packet()` 中 checksum 计算错误：

**错误代码（两个版本都有）**：
```python
calc_checksum = (~(sum(data_bytes) & 0xFF)) & 0xFF   # Bitwise NOT — 一补码
```

**ThinkGear 协议正确算法**：
```python
calc_checksum = (-sum(data_bytes)) & 0xFF              # 二补码 (0x100 - sum & 0xFF)
```

一补码（`~x`）与二补码（`-x`）对所有非零和始终差 1。例如：
- 数据和 = 0x85 → 设备发送 checksum = 0x7B
- 错误计算：`~0x85 & 0xFF = 0x7A` → **校验失败，丢弃**
- 正确计算：`-0x85 & 0xFF = 0x7B` → **校验通过，正常解析**

### 11.3 次要原因：未遍历多行数据包

原始代码只解析 `data_bytes[0] == 0x80`，但 BMD101 包常含多行：
```
[POOR_SIGNAL(0x02)] [RAW_WAVE(0x80)] [HEART_RATE(0x03)]
```
当 0x80 不是首行时 RAW_WAVE 被跳过。修复为 `while` 循环遍历。

### 11.4 修复文件

| 文件 | 修改内容 |
|------|----------|
| `ecg_intervention/ecg_biofeedback_psychopy.py` | `calc_checksum` 改为二补码；`_parse_packet` 改为遍历多行 |
| `ecg_intervention/ecg_biofeedback_training.py` | 同上 |

### 12.3 校验和一补码纠正

**第 6 轮修复将校验和改为了二补码 `(-sum) & 0xFF`，经实际硬件测试发现是错误的。**

实际 BMD101 发送的校验和是一补码 `(~sum) & 0xFF`：

以实际数据包为例：
```
AA AA 04 80 02 FE A2 DD
```
- 数据和 = `0x80 + 0x02 + 0xFE + 0xA2` = **0x222**
- 一补码 `(~0x22) & 0xFF = 0xDD` ✅ **与设备发送一致**
- 二补码 `(-0x222) & 0xFF = 0xDE` ❌ **差1，校验失败**

**修复文件：**
- `ecg_intervention/ecg_biofeedback_psychopy.py` — 改回 `(~sum(data_bytes)) & 0xFF`
- `ecg_intervention/ecg_biofeedback_training.py` — 同上

被试真实心率 ~70 BPM，CardioChip 程序显示 ~70 BPM 正确，但 biofeedback 程序显示 ~140 BPM（恰好 2 倍）。

### 20.2 根本原因：T 波被误检为 R 波（双倍触发）

**检测器**使用自适应阈值 `mean + 0.8 × std` 判断 R 波。
- R 波后 ~300ms 的 T 波幅度超过了此阈值
- 不应期仅 200ms，T 波在不应期之外
- 检测到的 RR 间期序列 ≈ [350ms (R→T), 507ms (T→R), ...]
- 平均 RR = 428ms → HR = 60/0.428 = **140 BPM**

**影响不仅是显示**：`r_peak` 也被 T 波误触发，导致试次计数翻倍，实验流程也被破坏。

### 20.3 修复一：提高 R 波检测阈值

`RPeakDetector.process()` 中：

| 文件 | 修改前 | 修改后 |
|------|--------|--------|
| `ecg_biofeedback_psychopy.py` | `mean + 0.8 * std` | `mean + 1.5 * std` |
| `ecg_biofeedback_training.py` | `mean + 0.8 * std` | `mean + 1.5 * std` |
| `ecg_biofeedback_practice.py` | `mean + 0.8 * std` | `mean + 1.5 * std` |

### 20.4 修复二：RR 间期逻辑验证（安全网）

在 `_parse_packet()` 中，R 波检测后新增验证：
- 建立运行中 RR 中位数（`median_rr`）
- 若新 RR 间期 `< 0.4 × median_rr`（例：857ms → 应 > 343ms），判定为 T 波误检
- 误检的 T 波不发射 `r_peak`，不更新 `_last_r_peak_time`
- 下一个真实 R 波跳过被跳过的 T 波，算出正确 RR 间期

**修改文件（均为 BMD101Worker / BMD101WorkerThread 内）**：
- `ecg_biofeedback_psychopy.py`
- `ecg_biofeedback_training.py`
- `ecg_biofeedback_practice.py`

### 20.5 修复三：改用 BMD101 芯片内置 HR（与 CardioChip 一致）

**关键发现**：从 Arduino 参考源码 `xindian.ino` 发现 BMD101 的 ThinkGear 协议中 `0x03` 代码上报芯片 DSP 自算心率。CardioChip 通过 `ThinkGear.dll` 读的就是这个值。

**修改前**：我们的程序完全忽略 `0x03`，仅靠自己做的 R 波检测算 HR（误差大）。

**修改后**：在 `_parse_packet()` 中新增 `0x03` HEART_RATE 解析，直接使用 BMD101 芯片算好的 HR：

```python
elif code == 0x03:
    if row_len >= 1:
        hr_val = data_bytes[i + 2]
        if 20 <= hr_val <= 250:
            self.queue.put(('heart_rate', float(hr_val)))
```

同时**移除**了原 RR 间期推算 HR 的代码（`self.queue.put(('heart_rate', round(hr, 1)))`），仅保留 R 波检测 + RR 间期验证用于试次计数。

## 21. 第二十一轮：结果页闪退修复 (2026-06-12)

### 21.1 问题
实验完成后，结果页（complete 状态）闪现不到一帧就退出，用户来不及阅读。

### 21.2 根因
`_handle_keys()` 中，complete 状态检测在 wait_key 状态检测之前：

```python
# Any key in complete state — clean exit
if self.controller.state == 'complete':
    self._running = False
    return

# Any key in wait_key state
if self.controller.state == 'wait_key':
    self.controller.handle_any_key(now)  # → _finish_experiment → state = 'complete'
```

触发流程：
1. 用户在 wait_key 状态按任意键 → `handle_any_key()` → `_finish_experiment()` → state = 'complete'
2. 当前帧绘图 + flip，结果页出现
3. **下一帧**：Windows 键盘连发或按键未释放 → `event.getKeys()` 再次读到同一按键
4. `state == 'complete'` → `_running = False` → 程序退出

结果页实际上只显示了不到 1 帧（~16ms），人眼完全看不到。

### 21.3 修复
在 `_finish_experiment()` 末尾调用 `event.clearEvents()`，清除残余按键缓冲区：

```python
def _finish_experiment(self, now):
    self._save_all_data()
    self.state = 'complete'
    self.instruction_text = self._build_results_text()
    self.feedback_type = ''
    self.feedback_text = ''
    event.clearEvents()  # ← 新增：防止同一按键连发触发退出
```

这样用户需要**松手后再按一个新键**才能退出结果页。

### 21.4 修改文件
| 文件 | 修改 |
|:----|:-----|
| `ecg_biofeedback_psychopy.py` | `_finish_experiment()` 末尾加 `event.clearEvents()` |
| `ecg_biofeedback_practice.py` | 同上 |

PyQt5 版（`ecg_biofeedback_training.py`）无此问题（Qt 事件系统自动处理连发）。

### 21.5 当前状态
- ✅ 结果页不再闪退，用户可从容阅读
- ✅ 阅读后按任意正常单次按键可退出

## 22. 第二十二轮：结果页闪退再次修复（最小展示时间保护）(2026-06-12)

### 22.1 问题
第 21 轮修复（`event.clearEvents()`）后，结果页仍会闪退。

### 22.2 根因分析

`event.clearEvents()` 在 PsychoPy 中不可靠——它只清除已缓冲但未读取的事件。Windows 键盘连发（key repeat）可能在 `clearEvents()` **之后**、下一帧 `getKeys()` **之前**产生新的按键事件：

```
Frame N:  key_down('space') → getKeys() → ['space'] → _finish_experiment() → clearEvents() → Draw(结果页) → Flip
Frame N+1: key_repeat('space') → getKeys() → ['space'] → state=='complete' → _running=False → 程序退出
```

结果页只能显示 ~1 帧（~16ms），人眼完全看不到。

### 22.3 修复：最小展示时间保护

不再依赖 `event.clearEvents()`，而是添加 **2 秒最小展示时间守卫**：

**`ExperimentController.__init__`** 新增字段：
```python
self._results_start_time = 0.0  # time.perf_counter when results page appeared
```

**`_finish_experiment()`** 中记录时间：
```python
self._results_start_time = now
```

**`_handle_keys()`** 中 `complete` 分支改为：
```python
if self.controller.state == 'complete':
    # 结果页至少展示 2 秒才允许退出
    if now - self.controller._results_start_time >= 2.0:
        self._running = False
    return  # 不到 2 秒时忽略所有按键
```

### 22.4 修改文件

| 文件 | 修改内容 |
|:----|:---------|
| `ecg_biofeedback_psychopy.py` | `__init__` 加 `_results_start_time`；`_finish_experiment()` 记录时间；`_handle_keys()` 加 2s 守卫 |
| `ecg_biofeedback_practice.py` | 同上三处 |

PyQt5 版（`ecg_biofeedback_training.py`）无此问题，无需修改。

### 22.5 当前状态
- ✅ 结果页至少显示 2 秒，不再因键盘连发闪退
- ✅ 2 秒后按任意键正常退出
- ✅ 不依赖 `event.clearEvents()` 的脆弱机制
